PHASE 3: PATTERN VALIDATION + SLF4J LOGGING
─────────────────────────────────────────────────────────────────────

📝 Changes to MeterReadings.java:

   BEFORE:
   ─────
   public record MeterReadings(
       @NotBlank 
       @Size(max = 64) 
       String smartMeterId,
       @NotEmpty List<ElectricityReading> electricityReadings)

   AFTER:
   ─────
   public record MeterReadings(
       @NotBlank 
       @Size(max = 64) 
       @Pattern(regexp = "^[a-zA-Z0-9-]+$", 
               message = "must contain only alphanumeric characters and hyphens")
       String smartMeterId,
       @NotEmpty @Size(max = 1000) List<@Valid ElectricityReading> electricityReadings)

Pattern Validation Examples:

   REJECTED (Special Characters):
   ❌ "smart-meter-0'; DROP--"         → SQL injection attempt
   ❌ "smart-meter-0@test"             → @ symbol
   ❌ "smart-meter#0"                  → # symbol
   ❌ "smart meter 0"                  → space
   ❌ "smart_meter_0"                  → underscore

   ACCEPTED (Valid Format):
   ✅ "smart-meter-0"                 → hyphen
   ✅ "smartmeter0"                   → alphanumeric only
   ✅ "SM-001-A"                      → mixed with hyphens
   ✅ "10101010"                      → numeric only
